# 自动化小说章节生成工作流 (Generate Chapter)

本工作流将使用多个 Agent 配合，快速、高保真地生成符合设定的硬核求生小说章节。
当用户通过 `/write_chapter [chapter_number] [plot_hints]` 命令触发时，必须严格依次执行以下步骤：

// turbo-all
1. **节点 1：前情读取（Context Loader）**
   - 读取上一章（即 `chapter_[chapter_number - 1].md`）的正文内容。
   - 读取 `/PROJECT_GUIDE.md` 了解全局网文排版与文风规则（务必牢记单句分段、去除AI味）。
   - 读取 `/docs/main_character.md` 获取江木峰的隐藏老书虫“肝帝/仓鼠”特质，以及史莱姆（咕叽）的反差萌与绝对忠诚防线。

2. **节点 2：大纲生成（Plot Writer）**
   - 目标：将上一步获取的上下文和用户提供的核心剧情提示 `[plot_hints]` 提供给大纲策划 Agent。
   - 动作：令 `@plot-writer` 生成包含核心冲突和人物动机的详细动作节拍表（Beat Sheet）。

3. **节点 3：正文起草（Chapter Drafter）**
   - 目标：基于生成的大纲节点，扩展正文细节。
   - 要求：强制 `@chapter-drafter` 输出约 3000 字正文。必须保持江木峰极致内敛的动作和带有烟火气、咸鱼风格的内心吐槽；史莱姆要有“咕叽”反差萌的互动。
   - 排版：应用“终极排版规则”：每一个句号结束后必须另起一行，严禁出现大段文字堆砌。

4. **节点 4：内部审查与重写循环（Critic & Rewrite Loop）**
   - 目标：执行自动化文稿核查机制。
   - 动作：将草稿传递给 `@continuity-checker`。
   - 审查要点：
     1. 是否出现系统播报中的违禁词汇（如“地狱级”、“无缝降临”等带有主观或网游夸张色彩的词）。
     2. 排版是否严格彻底遵循单句分段的大量留白网文格式。
     3. 是否准确演绎了江木峰内心吐槽与表面肝帝行动的强烈反差。
   - 循环协议：如果审查出任何不合格点，必须附带修改意见打回给 `@chapter-drafter` 进行定点强制重写。该重写审查循环最多进行 4 次。

5. **节点 5：输出定稿（Finalize）**
   - 目标：将完美通过审核的正文实体化。
   - 动作：以 `chapter_[chapter_number_padded].md`（如 `chapter_004.md`）的命名格式将定稿保存至 `/chapters/` 目录下。
   - 完成通知：向用户输出固定文案：“本章已生成并自动归档，请主编审阅。”
